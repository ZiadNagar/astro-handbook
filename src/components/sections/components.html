.astro Components

The .astro file is Astro's own component format. It borrows JSX-like syntax but compiles to pure HTML — no runtime framework required.

Anatomy of a .astro File

Every .astro file is split into two parts by a triple-dash fence (---):

src/components/Card.astro
---
// 1. COMPONENT SCRIPT (runs only at build/request time — server-side)
// This is where you import, fetch, define props, run logic

const { title, description } = Astro.props;
const date = new Date().toISOString();
---

<!-- 2. COMPONENT TEMPLATE (JSX-like HTML) -->
<div class="card">
  <h2>{title}</h2>
  <p>{description}</p>
  <small>{date}</small>
</div>

<!-- 3. OPTIONAL: scoped styles -->
<style>
  .card { border: 1px solid #ccc; padding: 1rem; }
</style>
⚡ Important
The component script runs at build time (or per-request in SSR). It's never in the browser. You can use await, call databases, read the file system — anything Node.js can do.

Expressions in Templates

---
const items = ['Apple', 'Banana', 'Cherry'];
const isVisible = true;
---

<!-- Rendering a list -->
<ul>
  {items.map((item) => (
    <li>{item}</li>
  ))}
</ul>

<!-- Conditional rendering -->
{isVisible && <p>I am visible</p>}
{isVisible ? <p>Yes</p> : <p>No</p>}

<!-- Set raw HTML (use carefully!) -->
<div set:html={rawHtmlString} />

Importing & Using Components

---
import Card from '../components/Card.astro';
import Button from '../components/Button.astro';
---

<Card title="Hello" description="World" />
<Button variant="primary">Click me</Button>
