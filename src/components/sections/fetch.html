Data Fetching

Since the component script runs on the server (or at build time), you can use await freely — no useEffect, no loading states needed for initial data.

Fetching External APIs

src/pages/posts.astro
---
// Top-level await works inside .astro files!
const response = await fetch('https://api.example.com/posts');
const posts = await response.json();
---

<ul>
  {posts.map((post) => (
    <li><a href={post.url}>{post.title}</a></li>
  ))}
</ul>

Reading Local Files

---
import fs from 'node:fs/promises';

const data = await fs.readFile('./data.json', 'utf-8');
const json = JSON.parse(data);
---

Astro.glob() — Import Multiple Files

Astro.glob() lets you import multiple files at once with a glob pattern — great for markdown posts, JSON files, etc.

---
// Import all markdown files from a folder
const posts = await Astro.glob('../content/blog/*.md');

// Sort by date descending
posts.sort((a, b) =>
  new Date(b.frontmatter.pubDate) - new Date(a.frontmatter.pubDate)
);
---

{posts.map((post) => (
  <a href={post.url}>{post.frontmatter.title}</a>
))}
ℹ️ Modern Alternative
Astro.glob() still works but Content Collections (next section) are the preferred, type-safe approach for managing content in Astro v2+.
