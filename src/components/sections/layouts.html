Layouts

Layouts are wrapper components that provide shared structure (header, footer, `<head>`) around page content via <slot />.

src/layouts/BaseLayout.astro
---
interface Props {
  title: string;
  description?: string;
}

const { title, description = 'My website' } = Astro.props;
---

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width" />
  <title>{title}</title>
  <meta name="description" content={description} />
</head>
<body>
  <header><nav>...</nav></header>

  <main>
    <slot />   <!-- page content renders here -->
  </main>

  <footer>...</footer>
</body>
</html>

Using a Layout in a Page

src/pages/about.astro
---
import BaseLayout from '../layouts/BaseLayout.astro';
---

<BaseLayout title="About Me" description="Who I am">
  <h1>About Me</h1>
  <p>I build stuff on the web.</p>
</BaseLayout>

Named Slots

Need multiple insertion points? Use named slots.

<!-- In layout: -->
<slot name="header" />   <!-- named slot -->
<slot />                 <!-- default slot -->
<slot name="footer" />   <!-- named slot -->

<!-- In page using the layout: -->
<MyLayout>
  <h1 slot="header">Page Title</h1>   <!-- goes to header slot -->
  <p>Body content</p>                 <!-- goes to default slot -->
</MyLayout>

Markdown Layouts

Markdown/MDX files can declare their layout in frontmatter.

src/content/blog/my-post.md
---
layout: ../../layouts/BlogPost.astro
title: My First Post
pubDate: 2024-01-01
---
