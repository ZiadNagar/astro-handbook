Content Collections v2+

Content Collections give you a type-safe, schema-validated way to manage local content — markdown, MDX, YAML, JSON. Think of it as a local database for your content.

1. Define Your Schema

src/content/config.ts
import { z, defineCollection } from 'astro:content';

const blogCollection = defineCollection({
  type: 'content',    // 'content' for md/mdx, 'data' for json/yaml
  schema: z.object({
    title: z.string(),
    pubDate: z.date(),
    author: z.string().default('Anonymous'),
    tags: z.array(z.string()).default([]),
    draft: z.boolean().default(false),
    cover: z.string().optional(),
  }),
});

export const collections = {
  'blog': blogCollection,
};

3. Query the Collection

src/pages/blog/index.astro
---
import { getCollection } from 'astro:content';

// getCollection returns typed, validated entries
const allPosts = await getCollection('blog');

// Filter out drafts
const published = allPosts.filter((post) => !post.data.draft);
---

{published.map((post) => (
  <article>
    <a href={`/blog/${post.slug}`}>{post.data.title}</a>
  </article>
))}

4. Render a Single Entry

src/pages/blog/[slug].astro
---
import { getCollection, getEntry } from 'astro:content';

export async function getStaticPaths() {
  const posts = await getCollection('blog');
  return posts.map((post) => ({
    params: { slug: post.slug },
    props: { post },
  }));
}

const { post } = Astro.props;
const { Content } = await post.render();  // renders markdown to component
---

<h1>{post.data.title}</h1>
<Content />   <!-- renders markdown body -->
