SSR & Output Modes

By default, Astro renders everything at build time (SSG). You can switch to server-side rendering per-page or for the whole project.

Three Output Modes

Mode Config Behavior
static output: 'static' All pages pre-rendered at build time (default)
server output: 'server' All pages rendered on each request
hybrid output: 'hybrid' Pages are static by default; opt pages into SSR

Switching to SSR

astro.config.mjs
import vercel from '@astrojs/vercel/serverless';

export default defineConfig({
  output: 'server',    // or 'hybrid'
  adapter: vercel(),  // deployment target adapter
});

Hybrid Mode — Opt-in SSR per Page

src/pages/dashboard.astro
---
// In 'hybrid' mode, opt this page into SSR:
export const prerender = false;  // SSR
// export const prerender = true;   // pre-render (default in hybrid)

// In 'server' mode, opt into static:
export const prerender = true;
---

Reading Request Data in SSR

---
export const prerender = false;

const { request, url, cookies, redirect } = Astro;

// Read URL params
const id = url.searchParams.get('id');

// Read cookies
const token = cookies.get('auth_token')?.value;

// Redirect
if (!token) return redirect('/login');

// Read POST body
if (request.method === 'POST') {
  const body = await request.json();
}
