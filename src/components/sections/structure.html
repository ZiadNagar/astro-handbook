03 · Project Structure

Project Structure

Astro enforces a specific directory convention. Understanding it upfront saves confusion later.

🔑 Key Rule
Only files inside src/pages/ become routable URLs. Everything else in src/ is imported but never exposed directly.

astro.config.mjs

This is the heart of your Astro project. All integrations, output mode, and base path live here.

astro.config.mjs
import { defineConfig } from 'astro/config';
import react from '@astrojs/react';
import tailwind from '@astrojs/tailwind';

export default defineConfig({
  site: 'https://mysite.com',   // needed for sitemaps, canonical URLs
  base: '/',
  output: 'static',             // 'static' | 'server' | 'hybrid'
  integrations: [react(), tailwind()],
  vite: {
    // pass-through Vite config here
  }
});
