Styling

Scoped styles are the default in .astro files. They only apply to elements in that component.

<h1>Only this h1 gets styled</h1>

<style>
  /* Automatically scoped — won't leak */
  h1 {
    color: red;
    font-size: 2rem;
  }
</style>

Use is:global or a separate CSS file imported in your layout for truly global styles.

<!-- Option 1: is:global modifier -->
<style is:global>
  body { margin: 0; font-family: sans-serif; }
</style>

<!-- Option 2: Import CSS file -->
---
import '../styles/global.css';
---

Install Tailwind with one command. Astro's integration auto-configures everything.

npx astro add tailwind

# Then just use classes in your templates:
<button class="bg-teal-500 hover:bg-teal-600 text-white px-4 py-2 rounded">
  Click Me
</button>

CSS Modules work via Vite, producing locally-scoped class names.

src/components/Button.module.css
.button { background: teal; padding: 8px 16px; }
src/components/Button.astro
---
import styles from './Button.module.css';
---
<button class={styles.button}><slot /></button>

Dynamic Class Names

---
const { variant = 'default' } = Astro.props;
---

<!-- Using template literals -->
<div class={`card card-${variant}`}>...</div>

<!-- Using class:list directive (Astro built-in) -->
<div class:list={['card', { 'card-active': isActive }, variant]}>...</div>
