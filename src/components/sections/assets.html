Assets & Images

Astro has a built-in image optimization pipeline. Images in src/assets/ are processed; images in public/ are served as-is.

The <Image /> Component

---
import { Image } from 'astro:assets';
import hero from '../assets/hero.jpg';
---

<!-- Astro auto-optimizes: WebP, sizing, lazy loading -->
<Image
  src={hero}
  alt="Hero image"
  width={800}
  height={400}
  format="webp"
  quality={80}
/>

The <Picture /> Component

---
import { Picture } from 'astro:assets';
import photo from '../assets/photo.jpg';
---

<Picture
  src={photo}
  alt="Responsive photo"
  formats={['avif', 'webp']}
  widths={[400, 800, 1200]}
/>
src/assets/ (Processed)

Images here are imported and processed by Astro. They get optimized, converted to WebP/AVIF, and get hashed filenames for caching.

public/ (Pass-through)

Files here are copied as-is. Use for favicons, robots.txt, OG images that need a fixed, predictable URL.
