07 · Islands Architecture

Islands Architecture

This is Astro's most important concept. Most of your page ships as zero-JavaScript HTML. Only the parts that need interactivity — Islands — load their framework and hydrate in the browser.

Client Directives — How to Hydrate

Add a client:* directive to any component to make it an Island. Without it, a React/Vue component renders as static HTML only.

Directive When JS Loads Use Case
client:load Immediately on page load Visible, critical UI
client:idle When browser is idle Less critical, below fold
client:visible When component enters viewport Lazy-loaded, far down page
client:media="(query)" When media query matches Mobile-only nav, etc.
client:only="react"` Browser only (no SSR) Components using browser APIs 
---
import Search from './Search.jsx';      // React component
import Counter from './Counter.svelte';  // Svelte component
import Chart from './Chart.vue';        // Vue component
---

<!-- No directive = static HTML, no JS -->
<Search />

<!-- Hydrate immediately -->
<Search client:load />

<!-- Wait for idle -->
<Counter client:idle />

<!-- Only when in viewport -->
<Chart client:visible />

<!-- Mobile only -->
<MobileNav client:media="(max-width: 768px)" />
⚠️ Islands are Isolated
Islands don't share state with each other by default. If you need shared state across islands, use a framework-agnostic store like nanostores (officially recommended by Astro).
