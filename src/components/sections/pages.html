05 · Pages & Routing

Pages & Routing

Astro uses file-based routing. Any file in src/pages/ with a supported extension becomes a URL.

Supported Page Formats

File URL Notes
src/pages/index.astro / Home page
src/pages/about.astro /about
src/pages/blog/index.astro /blog Folder index
src/pages/blog/[slug].astro /blog/anything Dynamic route
src/pages/[...all].astro Catch-all Rest params
src/pages/rss.xml.js /rss.xml API / feed endpoint

Dynamic Routes — getStaticPaths()

For dynamic routes in static mode, you must export getStaticPaths() to tell Astro which URLs to generate.

src/pages/blog/[slug].astro
---
export async function getStaticPaths() {
  // return every possible [slug] value
  return [
    { params: { slug: 'first-post' }, props: { title: 'First Post' } },
    { params: { slug: 'second-post' }, props: { title: 'Second Post' } },
  ];
}

const { slug } = Astro.params;
const { title } = Astro.props;
---

<h1>{title}</h1>
<p>Slug: {slug}</p>

API Endpoints

Files ending in .js or .ts in pages/ become API endpoints (JSON, XML, RSS feeds).

src/pages/api/hello.json.ts
import type { APIRoute } from 'astro';

export const GET: APIRoute = ({ params, request }) => {
  return new Response(
    JSON.stringify({ message: 'Hello, World!' }),
    { headers: { 'Content-Type': 'application/json' } }
  );
};
💡 Tip
You can also export POST, PUT, DELETE, PATCH from the same endpoint file.
